Registrar Lock Status

The system checks if the domain has the status `clientTransferProhibited`. If locked, the UI will clearly state: "This domain is locked. Please log in to your current registrar and unlock the domain before proceeding," and provide a link to a relevant help article.

Domain Age

Per ICANN rules, domains cannot be transferred within 60 days of registration or a previous transfer. If ineligible, the system will inform the user of the exact date the domain can be moved.

Privacy Services

While not always a blocker, active WHOIS privacy services can sometimes interfere. The system will flag this with a recommendation: "For a smoother transfer, we recommend disabling this with your current registrar temporarily."

Redemption Period

If a domain has expired and is in its "redemption period," it cannot be transferred. The system will detect this and instruct the user to first renew the domain with their current registrar.

Phase 2: The Automated Transfer Workflow

Once a domain passes the eligibility checks, the client moves through a simple, guided workflow. This timeline illustrates the core steps from initiation to our backend systems taking over the request.

1. Initiation

The client enters a single domain or uploads a bulk CSV file. The Pre-Transfer Eligibility Checker from Phase 1 confirms the domains are ready to move.

2. Authorization & Payment

The client provides the Authorization/EPP code. The system clearly explains the mandatory 1-year registration extension and its cost, then guides them through payment.

3. Final Confirmation

A final summary screen is presented for review. Upon confirmation, the user is notified that the automated transfer process will now begin.

4. Backend Automation

Our system makes an API call to the central domain registry, formally initiating the transfer. The customer's dashboard is updated in real-time to "Transfer in Progress."

Phase 5: Post-Transfer & Onboarding

A successful transfer is just the beginning. Our goal is to make clients successful on our platform immediately. We'll achieve this through an optional DNS record scan and a clear "Getting Started" checklist. Try it out below!

Automated DNS Record Import

To minimize downtime, our system will offer to scan the domain's previous nameservers to find existing DNS records (A, CNAME, MX, etc.).

It will then present these records to the user for a one-click import, ensuring a seamless transition for their website and email services.
